手把手带你配置VisuMZ Core Engine

发表于2026-01-21|更新于2026-01-21|游戏开发带你配置VisuMZ核心插件
|总字数:2.9k|阅读时长:10分钟|浏览量:

前言

不错，这次又是一个大坑，从一个新的空工程开始带你配置VisuMZ系列的全部核心插件，不过作为个人笔记的成分比较多，因为你直接让我进插件汉化说实话太麻烦了，还不如边看边写笔记（

插件的使用和安装方法默认读者全部都会了，不懂就参考Yanfly.moe Wiki

建议搭配官方范例工程食用

准备工作

本教程直接从一个空的工程开始，直接把插件全部倒进来

然后双击插件就能看到茫茫多的参数了，别急，我们一个一个来配置

Quality of Life Setting 游戏体验设置

光是这一个部分就有这么多可以配置的地方了，别急，一部分一部分来

加粗部分是我个人建议要注意的设置

Play Test 游戏测试

这里是游戏测试启动时的设置

New Game on Boot：是否在游戏测试中自动开始一个新的游戏（就是开一个新档），默认为false

No Play Test Mode：是否禁用测试模式

Open Console on Boot：启动时是否打开控制台，其实编辑器里也能设置这个，应该是插件设置的优先级更高

F6: Toggle Sound：是否开启F6切换静音模式

F7: Toggle Fast Mode：是否开启F7切换加速模式

CTRL + n: Quick Load: 是否开启ctrl+数字键快速读档，不算自动存档

NewGame > CommonEvent：设置游戏测试开始时运行的公共事件

Battle Test 战斗测试

这里是在编辑器进行战斗测试的设置

Add Item Type —Add Armor Type：道具、武器和防具每样都给一个

Added Quantity：如果勾选了Add Item Type，这里是给的数量

Shift+R: Recover All：对应快捷键完全恢复

Shift+T: Full TP：回满TP

Digit Grouping数字分组

就是类似于1234567 ➡️ 1,234,567这样的

Standard Text：就是输出的标准字符

Ex Text：通过drawTextEx输出的标准字符（比如对话框之类的）

Damage Sprites、Gauge Sprite：伤害数字和血条数字

可以设置这些地方是否要用数字分组

Country Locale：使用对应国家地区的分组方法，比如咱们是zh-cn，那就是四个为一组

Player Benefit 游玩优化

一系列优化玩家游玩体验的设置

最小遇敌率：由于rm的暗雷系统比较狂野，所以可能刚遇敌后走了两步又来一个，这里可以设置玩家暗雷遇敌需要的最少步数

Escape Alway：只要允许玩家逃跑，那么逃跑率永远为100%，无视速度之类的属性

Accuracy Formula：设置为visu的命中率公式，具体自己看插件。总之用就对了

Boost Accuracy：提高命中和闪避，用就对了

Level Up - Full HP/MP：升级自动回满血

Picture-Related 图片相关

Anti-Zoom Picture：避免图片被拉伸缩放导致图片质量下降

Picture Containers：在战斗和地图场景，是否将图片容器与SpriteSet分离，保持默认就行了

Misc 杂项

Ani：Mirror Offset：动画使用镜像效果时（这个在动画的注释里设置），是否将动画的x坐标也进行镜像

Auto-Stretch：是否自动拉伸游戏画面来匹配窗口大小，保持默认就行了

Font Shadows：用文字投影效果来代替原先的轮廓线效果

Font Smoothing：开启文字平滑（说实话感觉像素字体用这个没什么用。。）

Font Width Fix：修复非半角字符可能会有的一些bug，开就对了

Key Item Protection：保护关键道具不被卖出

Map Name Text Code：启用后，地图名称也可以使用控制符（就是\c[2]这种）

Modern Controls：可以使用Home/End键，开就对了

MV Animation Rate：使用MV格式创建动画时，动画播放的速率，默认为4，数字越低越快

NewGamer > CommonEvent：新游戏开始时自动运行的公共事件，不限于游戏测试环境

No Tile Shadows：关闭编辑器自动生成的图块阴影

Pixel Image Rendering：像素游戏必备，开启像素渲染

Require focus：游戏窗口不在最前时暂停运行

Shortcut Script：不懂，过，这里贴一下原文档的解释

Smart Event Collision：让事件只会和与优先级”和玩家同层“的东西碰撞，开就对了

Subfolder Name Purge：开着就对了

Battle System 战斗系统

设置使用的战斗系统，如果用了visu付费的战斗系统就要用这个设置，没有的话放着就行

Color Setting 颜色设置

设置游戏里各种东西的颜色，包括血条等等，这里也可以不急着弄

**注意！**如果想关掉outline，直接改mz自己的js是行不通的（因为被这个插件覆盖了），需要在这一项找到Alpha Color - Windows Font Outline，把最后一个数字改成0（也就是不透明度为0），就等于是关闭描边了

Gold Setting 金钱设置

对游戏内金钱进行设置，分别是：最大值、字体大小、图标、占位符（就是当金钱太多窗口显示不下的时候显示的文本）

Item Style：像绘制道具一样来绘制金钱（就是可以有图标、标签和值），总之开就对了

Image Loading 加载图片

在游戏启动时就进行预加载的图片，按需配置，一般没什么高要求的话就不用管了

Keyboard Input 键盘输入

WASD Movement：使用wasd来进行移动，这个按需设置就行

R Button：Dash Toggle：用R键来切换始终冲刺的状态

Name Input，有关输入名称界面的功能

  • Enable？：是否启用
  • Default Mode：默认的输入模式，用键盘或者用默认的方式（不过键盘输入似乎只能输英文）
  • QWERTY Layout：如果用默认输入模式，就将字母表按键盘的布局显示
  • Keyboard Message：在输入框中显示的文本（就是类似请输入文本这样的）
  • Banned Words：禁止词，应该没人用这个吧。。。

Number Input：可以在输入数字的时候使用键盘，不过只支持英语输入法

Button Assist：在一些界面上方显示的按钮说明

  • Finish Entry，Page Change, Switch To Keyboard, Switch to Manual
  • 不太懂，总之先不管也行吧。。

Menu Background Settings 菜单背景设置

设置在菜单背景显示的图片

Menu Button Assist Window 按钮说明设置

设置按钮说明的显示，这里就不说明了，因为我一般都是直接关了的。。

另外也可以适配手柄

Menu Layout Settings 菜单布局设置

可以分别对菜单的各Scene进行设置，有把握的可以试试，我一般也不管。。

Parameter Setting 参数设置

设置游戏里的各种数值

Displayed Parameters：在游戏中显示的属性

Extended Parameters：在visu插件中显示的其他属性

Show Actor Levels？在游戏中大部分地方显示角色的等级

Convert JS To Base：不懂，过

HP Crisis Rate：HP低到这里设置的比例后进入危机状态

JS Formula：过

Parameter Caps：分别设置角色和敌人各属性值的上限

X Parameter：对应编辑器里的追加能力值，下面是设置这些属性在游戏里显示的词汇，看不懂的缩写可以参考

S Parameter：对应特殊能力值

Icon：在各个能力值前绘制的图标，当然我都是直接不启用的

Custom Parameters 自定义参数

这里可以设置一些自定义的在游戏中显示的参数，这里自己参考着来吧，不过其实也不用管

因为这里的参数只能靠游戏内的数值（比如角色攻击、防御之类的）计算出来，基本上是只能看，没法直接来进行计算，所以感觉有点鸡肋，也可能是我不会用

Screen Resolution Setting 屏幕分辨率设置

基本上放着就行，要设置分辨率就在编辑器里弄就行

Screen Shake Setting 屏幕震动设置

可以给屏幕震动设置不同的效果，这个也是放着就行了。。

Title Command List 标题指令列表

在标题画面显示的指令，不动也行，或者也可以加一个跳转到自己的网站之类的

Title Picture Button 标题图片按钮

在标题画面显示的图片按钮，可以代替上一项中的指令使用

不过甚至可以在标题右下角显示跳转sns的图标，可以上wiki自己去找找教程

UI Setting UI设置

对UI进行设置

UI Area

  • Fade Speed：淡出淡入的速度
  • Box Margin：外边距
  • Command Window Width：指令窗口的宽度
  • Bottom Help Window：按键说明窗口启用，我一般是直接关了的
  • Right Aligned Menus：启用右对齐菜单，关掉的话就在左边
  • Show Buttons：选择是否在屏幕直接显示取消、菜单、翻页和数字按钮，基本是给移动端用的

Button Area Height：按钮空间高度

Bottom Button：让按钮放到底部

Side Button：将按钮放到UI侧边

State Icon Non-Frame：不懂，放着

Level - EXP Gauge：在等级下面画一个经验条

Parameter Arrorw：放着就行

Text Code Support：设置在技能名和角色昵称是否可以用文本控制符

Window Settings 窗口设置

对游戏内设置的窗口进行设置。我已经力竭了，懒得打原文了，你们自己对照着看吧。。

Defaults

  • Enable Masking：启用窗口遮挡（但是启用的话会导致窗口内容被挡住，所以还是关着吧）
  • correct skin bleed：窗口皮肤出血线，开着就行
  • line height： 行高
  • item padding：内边距
  • Back opacity：背景透明度
  • Translucent Opacity：半透明窗口的透明度
  • Window opening speed：窗口打开速度
  • column spacing：选择窗口的默认列间距
  • row spacing：选择窗口的行间距

scroll bar

  • show scroll bar？是否显示滚动条
  • thickness：厚度
  • offset：滚动条偏移量
  • bar body color：滚动条颜色
  • off bar color：滚动条背景颜色
  • off bar color：背景透明度

Selectable Items

  • show background？：显示背景
  • item height padding：垂直方向的内间距
  • JS：draw background：过

text poup window

  • duration per text：每个文本增加的持续时间
  • minimum duration：最小持续时间
  • maximum duration：最大持续时间

JS ：Quick Function 快捷JS函数

可以在这里设置能够在工程里快捷使用的JS函数，我不会用，就不管了。。。
